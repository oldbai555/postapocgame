## 前端布局 / 菜单 / 按钮样式问题 —— 提问提示词模板

这份文档用于在你遇到前端页面布局、左侧菜单、按钮样式、组件不显示等问题时，快速、准确地向 AI 描述问题，方便高效排查。

> 使用方式：遇到问题时，复制下面的模板，在合适的地方填写实际信息后发给 AI 助手即可。

---

### 一、标准提问模板（推荐直接复制）

```text
【页面路径】
- 当前访问路径（URL）：______

【期望效果】
- 我期望看到：______
  （例如：左侧应该有系统管理/角色管理/权限管理菜单；按钮应该是 Element Plus primary 样式并带图标等）

【实际表现】
- 实际看到：______
  （例如：左侧菜单区域是空白；按钮有文字但样式像原生 button；布局错位/内容被遮挡等）

【浏览器 Console 报错】
- 有 / 没有 报错
- 如果有，请贴出关键报错（可以复制文本或截取 1–2 行核心信息）：______

【Network 请求情况】
- /api/v1/auth/profile 状态码：____，返回大致内容：____
- /api/v1/menus/tree 或 /api/v1/menus/my-tree 状态码：____，返回大致内容：____

【相关代码文件】
- 我认为可能相关的文件路径：
  - ______
  - ______
  - ______

【最近自己做过的改动（如果有）】
- ______
```

---

### 二、信息说明与填写指引

- **【页面路径】**
  - 填写浏览器地址栏中当前访问的路径，例如：
    - `/login`
    - `/dashboard`
    - `/system/role`
    - `/system/permission`

- **【期望效果】**
  - 用 1–3 句话描述你“希望看到”的内容或行为，例如：
    - “左侧菜单应该展示系统管理、角色管理、权限管理三个一级菜单”
    - “点击登录后应该跳转到主布局，左侧出现菜单，右侧是内容区域”
    - “这个按钮应该是蓝色的 primary 按钮，并且 hover 时有颜色变化”

- **【实际表现】**
  - 描述你“现在实际看到的”情况，例如：
    - “左侧菜单区域是空白”
    - “菜单有数据，但都是一条竖线，没有文字”
    - “按钮只是灰色矩形，没有 Element Plus 样式”
    - “内容区域被顶部导航条遮住了一部分”

- **【浏览器 Console 报错】**
  - 打开 DevTools → Console，如果有红色错误，尽量复制出关键错误信息，例如：
    - `Uncaught (in promise) Error: 请求失败`
    - `TypeError: Cannot read properties of undefined (reading 'map')`
    - `Failed to resolve component: ElMenu`

- **【Network 请求情况】**
  - 打开 DevTools → Network，找到对应接口（通常是 `auth/profile`、`menus/tree`、`menus/my-tree`），关注：
    - **Status**：200 / 400 / 401 / 500 等
    - **Response Body**：复制核心 JSON 部分，例如：
      - `{"code":0,"msg":"OK","data":{"list":[...]}}`
      - `{"code":401,"msg":"未提供认证信息"}`
  - 这些信息用于判断是接口本身报错，还是前端渲染 / 路由逻辑问题。

- **【相关代码文件】**
  - 至少列出 2–3 个你认为“和这个问题最相关”的前端文件路径，例如：
    - `admin-frontend/src/layouts/DefaultLayout.vue`
    - `admin-frontend/src/components/layout/SideMenu.vue`
    - `admin-frontend/src/stores/user.ts`
    - `admin-frontend/src/router/index.ts`

- **【最近自己做过的改动】**
  - 如果你刚刚对某些文件做了修改，可以简单说明，例如：
    - “我把菜单数据从 `/menus/tree` 改成了 `/menus/my-tree`”
    - “我调整了 DefaultLayout.vue 里的 `el-container` 结构”
    - “我改动了 `SideMenu.vue` 中 `el-menu-item` 上的 `:index` 绑定”

### 三、完整示例一：菜单不显示

下面是一个「左侧菜单不显示」的完整提问示例，你可以直接参考格式：

```text
【页面路径】
- 当前访问路径（URL）：/dashboard

【期望效果】
- 登录成功后，应该跳转到主布局，左侧显示系统管理相关菜单（角色管理、权限管理、部门管理等）。

【实际表现】
- 登录成功后 URL 已变成 /dashboard，但左侧菜单区域完全空白，只看到顶部导航栏和空白内容区。

【浏览器 Console 报错】
- 有报错
- 报错内容：
  - Uncaught (in promise) Error: 请求失败
  - at src/stores/user.ts:55 （fetchMenus 调用位置）

【Network 请求情况】
- /api/v1/auth/profile 状态码：200，返回内容类似：{"code":0,"msg":"OK","data":{"id":1,"username":"oldbai"}}
- /api/v1/menus/tree 状态码：200，返回内容类似：{"code":0,"msg":"OK","data":{"list":[{"id":1,"name":"系统管理","children":[...]}]}}

【相关代码文件】
- admin-frontend/src/layouts/DefaultLayout.vue
- admin-frontend/src/components/layout/SideMenu.vue
- admin-frontend/src/stores/user.ts
- admin-frontend/src/router/index.ts

【最近自己做过的改动（如果有）】
- 我刚刚把菜单接口从 /api/v1/menus/my-tree 改成了 /api/v1/menus/tree
```

### 四、完整示例二：按钮样式异常

```text
【页面路径】
- 当前访问路径（URL）：/system/role

【期望效果】
- 角色列表页顶部有一个“新增角色”的蓝色 primary 按钮，使用 Element Plus 的 <el-button type="primary"> 样式。

【实际表现】
- 现在页面上有一个“新增角色”的按钮，但样式是灰色原生按钮，没有 Element Plus 样式，hover 也没有任何效果。

【浏览器 Console 报错】
- 有报错
- 报错内容：
  - [Vue warn]: Failed to resolve component: ElButton

【Network 请求情况】
- /api/v1/auth/profile 状态码：200
- /api/v1/roles 状态码：200，返回列表正常

【相关代码文件】
- admin-frontend/src/views/system/RoleList.vue
- admin-frontend/src/main.ts

【最近自己做过的改动（如果有）】
- 我之前去掉了 main.ts 里的 Element Plus 全局注册，后面又加回来，但不确定是否正确：
  - app.use(ElementPlus)
```

---

有了这些信息，AI 助手可以快速判断问题属于：

- 接口 / 权限 / Token 问题
- Store / 缓存 / 动态路由注入问题
- 布局组件（如 `DefaultLayout`、`SideMenu`）写法或样式问题
- 组件库未正确注册（Element Plus）导致的渲染问题

从而尽快给出针对性的修改建议或直接帮你改代码。
